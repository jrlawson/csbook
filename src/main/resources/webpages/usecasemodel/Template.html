Use Case Name: Use Case Name

User Story:

Brief description of the use case, in non-technical terms. The narrative should describe why the actors want to do this use case. What grabs them, motivates them, and why they'd pay to do this.

Success Criteria:

Things that need to happen to deliver value to stakeholders. For instance, in the case of an Automatic Teller Machine, the success criteria for a “withdraw cash” use case might be (from the account holder’s perspective) “the correct amount of cash was provided to the account holder” These success criteria need to fall within the scope of the system. If they fall outside the scope of this system they are not success criteria for this use case, or the use case may be part of another system.

Minimum Requirements:

Things that need to be guaranteed (to a participating stakeholder) even if we fail to meet success criteria. Guarantees that the stakeholder’s need before they even attempt the use case. E.g. from the bank’s perspective, “If the account holder gets cash, the account will be debited by that amount”, or from the account holder’s perspective, “if the account is debited, the amount of the debit is provided in cash”

Preconditions:

  • Things that need to be in place before the use case starts.

Post-conditions:

  • Things that need to be in place when the use case ends (e.g. transactional semantics).

Actors:

  • Primary— Instantiates the use case. Usually, but not always a stakeholder
  • Secondary— Becomes involved with the system later (optional).
  • Off stage— Holds a stake without interacting with the system (optional).

Mapping: Stakeholders -> Value Upon Success:

Abstraction Level:

  • Cloud : “Product vision”, for instance “Forever free the user from the drudgery of housework”
  • Kite: “Conceptual”, for instance “Self-cleaning oven”
  • Surface : A use case that makes sense to a user
  • Fish : A use case that has value to an implementer of the system but may lack relevance for its user (few of these)
  • Clam: Fine-grain detail use case, for instance where precise definition of an algorithm is important (very few of these)
